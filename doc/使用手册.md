## 1. 快速入门

### 启动应用

```bash
# 命令行启动
python -m py_project.main

# 或使用注册的入口命令
py-knowledge
```

启动后应用会自动创建一个空白笔记，即可开始编辑。

### 基本工作流

1. **Ctrl+N** — 新建笔记
2. 输入 Markdown 内容 → 右侧实时预览
3. **Ctrl+S** — 保存（自动创建版本快照 + 更新搜索索引 + 解析链接）
4. 在搜索面板查找内容，在标签面板整理分类

### 面板说明

## 3. 笔记管理

### 新建笔记

- 菜单：**File → New Note** (`Ctrl+N`)
- 新建的笔记标题默认为 "Untitled"

### 保存笔记

- 菜单：**File → Save** (`Ctrl+S`)
- 保存时自动执行：版本快照 → FTS 索引更新 → 链接解析
- 每按一次 Ctrl+S 记录一个版本

### 打开笔记

- 双击 Notes 面板中的笔记
- 从搜索结果中点击
- 从反向链接面板中点击

### 重命名笔记

- 菜单：**Note → Rename Note** (`F2`)
- 弹出输入框，输入新标题

### 删除笔记

- 菜单：**Note → Delete Note** (`Ctrl+D`)
- 确认对话框确认后删除（软删除，存储在数据库中可恢复）

## 4. 编辑器与 Markdown

### 编辑区（左侧）

- 纯文本 Markdown 编辑
- **语法高亮**：标题、粗体/斜体、代码块、链接、wikilink、数学公式等 17 种规则
- **行号显示**：左侧边栏显示行号
- **当前行高亮**：光标所在行背景加深
- **代码块多行高亮**：围栏代码块（`` ``` ``）整体高亮

### 预览区（右侧）

- 实时 HTML 渲染预览
- **300ms 防抖**：停止输入 300ms 后自动刷新预览
- 支持的 Markdown 元素：

| 元素 | 语法 |
|---|---|
| 标题 | `# H1` `## H2` ... `###### H6` |
| 粗体 | `**bold**` |
| 斜体 | `*italic*` |
| 粗斜体 | `***bold italic***` |
| 内联代码 | `` `code` `` |
| 代码块 | ` ```python ... ``` ` |
| 表格 | `\| A \| B \|` |
| 无序列表 | `- item` `* item` `+ item` |
| 有序列表 | `1. item` |
| 引用块 | `> quote` |
| 分割线 | `---` `***` `___` |
| 数学块 | `$$ ... $$` |
| 内联数学 | `$...$` |
| 图片 | `![alt](url)` |
| 外部链接 | `[text](url)` |

### 缩放

- **Ctrl+=** 放大字体
- **Ctrl+-** 缩小字体

## 5. 标签系统

### 标签树

左侧 Tags 面板以树形结构展示所有标签，支持**无限层级嵌套**。

### 添加标签

1. 右键点击 Tags 面板中的某个标签（或根节点）
2. 选择添加子标签
3. 输入标签名称和颜色
4. 或使用菜单：**Note → Add Tag** (`Ctrl+T`)

### 路径批量创建

标签服务支持 `/` 路径创建，例如 `python/数据分析/pandas` 会一次性创建三个层级标签。

### 为笔记打标签

1. 打开目标笔记
2. 通过标签编辑对话框选择标签关联

### 按标签过滤笔记

**双击** Tag 面板中的标签 → Note 列表自动过滤为该标签下的笔记。

## 6. 全文搜索

### 基本搜索

1. 在右侧 Search 面板输入框输入关键词
2. 按 **Enter** 执行搜索
3. 结果列表显示：笔记标题 + 含 `<mark>` 高亮的上下文片段

### 高级搜索

搜索支持**特殊前缀**过滤：

| 语法 | 说明 | 示例 |
|---|---|---|
| `python django` | 搜索包含两个词（AND）的笔记 | `python django` |
| `tag:python` | 搜索标签为 "python" 的笔记 | `tag:python` |
| `title:笔记` | 搜索标题包含 "笔记" 的笔记 | `title:笔记` |
| 组合使用 | 前缀 + 关键词可组合 | `tag:python flask` |

### 搜索实现

基于 SQLite **FTS5** 全文搜索引擎，支持：
- Porter 词干提取（英文）
- unicode61 分词器
- 自动增量索引更新（Ctrl+S 时同步）

## 7. 版本历史

### 自动快照

每次 **Ctrl+S** 保存时自动创建版本快照：
- 存储笔记的**完整文本**
- 通过 SHA256 哈希**自动去重**（内容未变不产生新版本）
- 版本在 Versions 面板以列表展示

### 查看版本列表

- 菜单：**Version → View History** (`Ctrl+H`)
- 或直接在右侧 Versions 面板查看

### 对比版本

1. 在 Versions 面板中选择**两个版本**（扩展选择）
2. 点击 **Compare** 按钮
3. 弹出侧边双栏对比窗口：
   - 左栏：旧版本
   - 右栏：新版本
   - 差异段落高亮标注
   - 顶部显示变更统计

对比基于 **Myers 差分算法**，快速路径使用哈希匹配跳过相同段落。

### 恢复版本

1. 在 Versions 面板中选择一个版本
2. 点击 **Restore** 按钮
3. 确认对话框 → 编辑器内容替换为该版本

**注意：** 恢复前请确保已保存当前修改。

### 清理旧版本

- 菜单：**Version → Cleanup Old Versions**
- 保留策略：保留第一个、最后一个以及每第 10 个版本
- 会提示当前版本总数，确认后执行

## 8. 双向链接与块引用

### Wikilinks（笔记间链接）

在编辑器中输入 `[[笔记标题]]` 即可创建指向其他笔记的链接：

```markdown
参见 [[Python 学习笔记]] 中的相关内容。

或使用别名：[[Python 学习笔记|我的 Python 笔记]]
```

- 保存时自动解析链接目标
- 如果目标笔记存在 → 建立有向链接
- 如果目标笔记不存在 → 标记为未解析

### 反向链接

选中某篇笔记后，右侧 **Backlinks 面板**自动显示所有**引用了当前笔记**的其他笔记，形成知识网络。

### 块级引用

使用 `((block-id))` 语法可精确引用某段落：

```markdown
关于这一点，参见 ((a1b2c3d4e5f6))
```

每个段落（Block）有唯一的 12 位十六进制 ID（SHA256 前 12 位）。

## 9. 附件管理

### 添加附件

1. 在底部 **Attachments 面板**右键 → **Add File**
2. 选择本地文件
3. 文件被复制到应用数据目录 `attachments/年/月/uuid_名称`

支持的文件类型：
- 图片（自动获取尺寸）
- 文档
- 任意文件

### 插入附件引用

右键附件列表中的文件 → **Insert**：
- 图片 → 插入 `![文件名](attachment://id)`
- 其他 → 插入 `[文件名](attachment://id)`

### 管理附件

右键菜单：
- **Insert** — 在当前光标位置插入引用
- **Unlink** — 从当前笔记取消关联（不删除文件）
- **Delete** — 删除附件文件及记录

## 10. 截图标注

### 截图流程

1. 菜单：**Tools → Screenshot Capture** (`Ctrl+Shift+S`)
2. 进入截屏模式：

**Step 1 — 区域选择：**
- 半透明遮罩覆盖全屏
- 拖拽鼠标框选截图区域
- 选中的区域变为透明

**Step 2 — 标注（可选）：**

| 工具 | 操作 |
|---|---|
| **矩形** | 拖拽绘制矩形框 |
| **箭头** | 拖拽绘制箭头（自动渲染箭头尖端） |
| **文字** | 点击放置文字标签 |
| **颜色** | 标注前选择颜色 |

3. 点击完成 → 截图以 PNG 保存，标注信息以 JSON 存储
4. 自动在编辑器光标位置插入 `![截图.png](attachment://id)`

### 标注数据

标注数据以 JSON 格式存储在 `attachments` 表的 `annotation_json` 字段，包含矩形的坐标、箭头的起终点、文字的位置和内容。

## 11. 导入与导出

### 导入 Obsidian Vault

1. 菜单：**File → Import Obsidian**
2. 在对话框中：
   - 选择 Vault 目录路径
   - 点击 **Scan** 扫描目录中的 .md 文件
   - 预览列表显示文件名、标签、链接数量
   - 勾选要导入的笔记
   - 点击 **Import** 执行导入

导入时自动处理：
- **YAML Frontmatter** — 解析为标签和元数据
- **`[[wikilinks]]`** — 解析并建立链接关系
- **标签** — 自动创建对应的层级标签

### 导出 Markdown

1. 菜单：**File → Export Markdown**
2. 选择输出目录
3. 选择导出模式：
   - **Flat（扁平）** — 所有 .md 文件放在同一目录
   - **Tree（树形）** — 按标签层级创建子目录
4. 自动生成 YAML Frontmatter（标题、日期、标签）
5. `[[wikilinks]]` 自动转换为相对 `.md` 路径

### 导出 OPML（思维导图）

标题层级自动映射为 OPML 大纲树，可在 XMind、FreeMind、MindManager 等工具中打开。

## 12. 主题切换

### 切换主题

- 菜单：**View → Toggle Theme** (`Ctrl+Shift+T`)
- 暗色主题（Catppuccin Mocha 配色）←→ 亮色主题

### 影响范围

主题切换同时更新：
- 主窗口 Chrome（菜单栏、工具栏、状态栏、停靠面板）
- 编辑器区域（背景、文字、选区颜色）
- 预览区（通过 CSS 自定义属性重新渲染）
- 滚动条、分割器手柄等细节元素

## 14. 数据存储位置

### 数据库

SQLite 数据库文件存储在：

| 平台 | 路径 |
|---|---|
| Windows | `%APPDATA%\PyKnowledge\knowledge.db` |
| macOS | `~/Library/Application Support/PyKnowledge/knowledge.db` |
| Linux | `~/.local/share/PyKnowledge/knowledge.db` |

### 附件

```
<数据目录>/
├── knowledge.db           # SQLite 数据库（所有笔记、标签、版本）
└── attachments/           # 附件文件
    ├── 2025/
    │   ├── 01/
    │   │   └── <uuid>_<文件名>
    │   └── 02/
    └── ...
```

## 提示与技巧

1. **先保存再关闭** — 按 `Ctrl+S` 确保修改已持久化并生成版本快照
2. **善用搜索** — `tag:` 和 `title:` 前缀能精准定位笔记
3. **聚焦模式** — 按 `Ctrl+B` 隐藏所有面板，仅保留编辑器+预览
4. **定期清理版本** — 使用 Version → Cleanup 减少数据库体积
5. **知识网络** — 多用 `[[wikilinks]]` 在笔记间建立关联，通过反向链接发现隐藏的联系
6. **备份** — 定期备份 `knowledge.db` 和 `attachments/` 文件夹
